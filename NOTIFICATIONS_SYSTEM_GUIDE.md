# 🔔 Система уведомлений для запросов на отпуск и болничный

## Что было добавлено

### 1. **Звуковые уведомления** 🔊
Красивые оригинальные звуки генерируются через Web Audio API:
- **🏖️ Новый запрос на отпуск** - двухтональный звон
- **✅ Одобрено** - счастливый поднимающийся звук (три тона)
- **❌ Отклонено** - мягкое нежное уведомление

**Файл:** `src/lib/notificationSound.ts`

```typescript
playNotificationSound("new_request");   // При новом запросе
playNotificationSound("approved");      // При одобрении
playNotificationSound("rejected");      // При отклонении
```

### 2. **Мигающий счетчик на табе** ✨
Когда появляется новый непрочитанный запрос, счетчик начинает мигать красным:
- `[1] Leaves` - 1 непрочитанный запрос
- `[3] Leaves` - 3 непрочитанных запроса
- Мигает каждые 600ms для привлечения внимания
- Автоматически исчезает когда все прочитаны

### 3. **Баннер на странице Leaves** 📢
На странице `/leaves` админ видит красивый мигающий баннер если есть непрочитанные запросы:
```
🔔 3 pending requests awaiting review
```

### 4. **Система отслеживания прочитанности** 📖
Каждый запрос имеет статус `isRead`:
- `false` - запрос не прочитан админом  
- `true` - админ просмотрел запрос

При двойнм клике на запрос для одобрения/отклонения:
1. Запрос автоматически отмечается как прочитанный
2. Счетчик уменьшается на 1
3. Звуковое уведомление подтверждает действие

## Как это работает

### Для сотрудника (создание запроса)
1. Сотрудник кликает "Запросить отпуск"
2. При создании запроса `isRead: false`  
3. Уведомления отправляются всем админам организации
4. **Звук:** Красивый двухтональный звон

### Для администратора
1. Админ видит мигающий красный счетчик на табе `Leaves`
2. Кликает на `Leaves` - видит баннер и список запросов
3. **На странице:** Баннер мигает: "🔔 3 pending requests awaiting review"
4. Админ нажимает ✅ или ❌ на запрос
5. **Звук:** Счастливый звук для одобрения, мягкий для отклонения
6. **Счетчик:** Уменьшается на 1 (3→2)

## Новые функции Convex

### Запросы (Queries)
```typescript
// Получить количество непрочитанных запросов
api.leaves.getUnreadCount({ requesterId: userId })
// Возвращает: number
```

### Мутации (Mutations)
```typescript
// Отметить один запрос как прочитанный
api.leaves.markLeaveAsRead({ leaveId })

// Отметить все запросы организации как прочитанные
api.leaves.markAllLeavesAsRead({ requesterId })
```

## Интеграция с другими системами

### Система уведомлений (Notifications)
- При создании запроса создается уведомление с типом `"leave_request"`
- Категория: `"leave_approved"`, `"leave_rejected"`

### Sidebar
- На иконке `Leaves` появляется красный мигающий счетчик
- Счетчик видно только для админов/супервизоров
- Автоматически обновляется в реальном времени
